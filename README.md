# 🚀 Prueba de Integración Git — Sitio Web Estático

[![HTML5](https://img.shields.io/badge/HTML5-E34F26?style=flat-square&logo=html5&logoColor=white)](https://developer.mozilla.org/html)
[![Git](https://img.shields.io/badge/Git-F05032?style=flat-square&logo=git&logoColor=white)](https://git-scm.com)

Repositorio de validación del flujo de trabajo con Git y GitHub. Contiene un sitio web estático simple con página principal y página de contacto, utilizado para verificar el ciclo completo de `add → commit → push`.

## 🧠 Contexto Pedagógico y Teórico
Dominar el flujo de trabajo con Git es una habilidad no negociable en la ingeniería de software colaborativa. Este repositorio representa el punto de partida práctico: la creación de un repositorio desde cero, la vinculación con un remoto en GitHub, y la validación del flujo de publicación de cambios. Más allá de su simplicidad técnica, el valor de este repositorio es metodológico: establece el flujo de trabajo que se replicará en proyectos de mayor complejidad.

## ⚙️ Tecnologías y Metodologías Aplicadas
* **HTML5**: Páginas web estáticas con estructura semántica básica, suficientes para validar el ciclo de vida de Git sin añadir complejidad innecesaria al entorno de prueba.
* **Git + GitHub**: Prueba del flujo completo de control de versiones distribuido, incluyendo la configuración del remote, el commit inicial y el push a la rama principal.

## 🛠️ Desglose Técnico (El "Cómo")
* **`index.html`**: Página de inicio del sitio estático.
* **`contacto.html`**: Página de contacto, validando que el repositorio puede gestionar múltiples archivos HTML en el mismo directorio raíz.
